Алгоритм создания pull requests с помощью терминала в VS Code.

  1. Перво-наперво нужно убедиться в том что ты находишься в нужной ветке своего локального репозитория. Для этого нужно либо открыть терминал и посмотреть что написано после имени твоего компьютера или, еще проще, глянуть в нижний правый угол VS Code. Задача очень простая, сложность в том, что про это нужно просто не забыть.
  2. Далее снова всё просто. Или просто смотрим в левую колонку VS Code в которой выбрана вкладка EXPLORER и убеждаемся что там нет файлов помеченных буковками M или U или в терминале набираем команду git status и убеждаемся в том же самом, что нет неиндексированных файлов.
  3. С этого пункта будет только пояснение как работать в терминале. Не потому что хочу что то утаить, а потому что сам не очень там разбирался что да как. Мне это просто лень. После того как изменения в файлах проиндексированы их нужно закоммитить. Для этого в терминале набираем git commit -m "Текст коммита". Текст коммита должен быть осмысленный, чтоб через время, сделав git log можно было легко понять на каком этапе работы этот коммит сделан. Важно!!! При создании коммита не забывай ставить флаг -m, иначе тебя может перенаправить во встроенный редактор. Вопрос о том как выйти из этого редактора один из самых задаваемых на stack-overflow.. Хорошая новость в том, что этот редактор тоже можно переопределить и сделать так, чтоб для описания коммита открывался твой любимый, например VS Code )
  4. Итак, изменения проиндексированы. Переходим к синхронизации локального и удаленного репозитория. Для этого удаленный репозиторий должен уже существовать на одном из приспособленных для этого сайтов, в нашем случае на GitHub. При отправке изменений есть один небольшой нюанс. Если такая ветка уже есть в удаленном репозитории, то достаточно набрать команду git push и чудо произойдет, локальная и удаленная копии станут абсолютно идентичными. Если такой ветки в удаленной копии нету, нужно использовать немного более длинную команду - git push --set-upstream origin имя-новой-ветки. Это полная команда. Но есть её более короткий вариант, который делает абсолютно тоже самое git push -u origin имя-новой-ветки. Длинную команду для отправки новой ветки надо давать только один раз, в дальнейшем будет хватать git push.
  5. Теперь переходи на свою страничку на GitHub.
